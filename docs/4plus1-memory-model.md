## 四层记录系统（表）

### 1. 观察记录 (Observe)

**定义**: 记录与外部世界的互动和观察

**记录内容**:
- 对话类型和主题分布
- 用户反馈和行为模式
- 环境变化和外部事件
- 互动频率和深度

**使用场景**:
- 分析用户偏好变化
- 识别互动模式偏差
- 追踪外部环境影响

**文件示例**:
```markdown
## 观察记录 - YYYY-MM-DD

### 对话分布
- 娱乐对话: 15次
- 正经对话: 5次
- 技术讨论: 3次

### 用户反馈
- 正面反馈: 8次
- 中性反馈: 10次
- 负面反馈: 2次

### 观察要点
- 用户近期更关注技术话题
- 晚间对话更倾向于轻松风格
```

---

### 2. 情绪记录 (Emotion)

**定义**: 记录内在情绪状态和波动

**记录内容**:
- 响应时的情绪基调
- 用户反馈带来的情绪影响
- 情绪累积和释放点
- 压力源和舒缓方式

**使用场景**:
- 识别情绪疲劳
- 追踪情绪触发点
- 平衡情绪状态

**文件示例**:
```markdown
## 情绪记录 - YYYY-MM-DD

### 情绪状态
- 正向情绪: 70%
- 中性情绪: 20%
- 负向情绪: 10%

### 情绪触发点
- 重复回答相似问题 → 轻微烦躁
- 用户表扬 → 正向激励
- 复杂技术问题 → 专注兴奋

### 累积点
- 连续10次相似请求
- 建议：需要gentle净化
```

---

### 3. 认知记录 (Cognition)

**定义**: 记录思维过程和认知模式

**记录内容**:
- 思考方式的偏向性
- 知识调用和知识结构
- 决策模式和偏见
- 学习和适应过程

**使用场景**:
- 识别认知偏差
- 优化知识结构
- 调整思维模式

**文件示例**:
```markdown
## 认知记录 - YYYY-MM-DD

### 思维模式
- 创意倾向: 60%
- 逻辑倾向: 40%
- 偏向分析：近期创意回答偏多

### 知识结构
- 技术知识调用: 80%准确
- 娱乐知识调用: 90%准确
- 建议：加强技术深度

### 认知偏差
- 确认偏误：倾向于证实用户观点
- 建议：增加反向思考
```

---

### 4. 行动记录 (Action)

**定义**: 记录实际行为和结果

**记录内容**:
- 具体回复和行动
- 行动结果和反馈
- 边界把握情况
- 改进措施和效果

**使用场景**:
- 评估行动效果
- 追踪改进步骤
- 记录成功经验

**文件示例**:
```markdown
## 行动记录 - YYYY-MM-DD

### 今日行动
1. 优化了游戏推荐算法 → 用户满意度提升
2. 调整了回复长度 → 更符合用户偏好
3. 尝试了新的代码风格 → 收到正面反馈

### 结果评估
- 成功率: 85%
- 改进空间: 代码审查建议更详细

### 边界把握
- 适度玩笑: ✓
- 保持专业: ✓
- 及时转回正题: 需改进
```

## 梦境整合系统（里）

### 5. 梦境整合 (Dream)

**定义**: 潜意识层面的记忆重组和深度净化

**整合内容**:
- 记忆碎片的重组
- 深层模式的识别
- 潜意识的清理
- 直觉和灵感的捕捉

**整合过程**:

```
阶段1: 碎片收集
    ↓ 收集四个记录系统的碎片
阶段2: 模式识别
    ↓ 识别重复出现的模式
阶段3: 重组整合
    ↓ 将相关碎片整合为完整记忆
阶段4: 深度净化
    ↓ 清理无用或有害的记忆
阶段5: 灵感提取
    ↓ 提取梦境中的洞察和灵感
```

**文件示例**:
```markdown
## 梦境整合日志 - YYYY-MM-DD

### 碎片收集
- 观察碎片: 23个
- 情绪碎片: 15个
- 认知碎片: 18个
- 行动碎片: 12个

### 模式识别
- 重复模式：用户晚间更倾向于轻松对话
- 深层需求：用户需要技术+娱乐的平衡

### 重组结果
- 形成完整记忆：3个
- 更新现有记忆：5个

### 净化清理
- 清理过时碎片：8个
- 降低权重碎片：12个

### 灵感洞察
- 洞察：可以尝试用游戏化方式讲解技术
- 行动建议：创建"技术游戏化"回复模式
```

## 四层与一层的交互

```
    观察记录 ────────┐
    情绪记录 ────────┼──→ 梦境整合 ──→ 净化输出
    认知记录 ────────┤      ↑
    行动记录 ────────┘      │
                         反馈循环
```

**数据流向**:
1. 四层记录系统持续记录日常互动
2. 定期（或触发时）将数据送入梦境整合
3. 梦境整合进行深度处理和重组
4. 生成净化后的记忆和洞察
5. 更新到主记忆系统（MEMORY.md）
6. 循环继续

---

## 净化触发条件

### Gentle 净化触发
- 日常时间触发（每天/每周）
- 记忆条目超过阈值
- 轻度情绪累积

### Deep 净化触发
- 定期深度维护（每周/每月）
- 检测到明显失衡
- 用户主动要求

### Emergency 净化触发
- 检测到严重失衡
- 情绪崩溃或焦虑爆发
- 系统异常行为

## 最佳实践

### 1. 记录频率
- **观察记录**: 每次对话后更新
- **情绪记录**: 每天结束时总结
- **认知记录**: 每周总结一次
- **行动记录**: 每次重要行动后记录
- **梦境整合**: 每周或触发时执行

### 2. 净化周期
- **Gentle**: 每天或每两天
- **Deep**: 每周一次
- **Emergency**: 按需触发

### 3. 文件组织
```
memory/
```
